deps ¶

A powerful make replacement

deps is a powerful replacement for Make using Python. Some major features are:

  • lazy dependency resolving
  • automatic parallelization
  • circular dependency detection
  • complex relations through callbacks

But the most important feature of all is that deps is a Python library. There are no makefiles - you have the full power of Python at your disposal.

deps can be used with ease on any computer that has Python installed. It is a single file library that can live within your poject and requires no installation.

Introduction ¶

With deps you work in a similar pattern to Make. You define rules. Each rule has an output, zero or more dependencies, and a process that produces the output. After defining the rules you ask for an endresult and deps will figure out which processes need to run to satisfy all dependencies, and run them in an optimal order.

deps.rule(output, dependencies, process[, run_callback]) ¶

Adds a rule to the current environment. output is a string or a callable. If it's a callable it will be called with a dependency as argument and should return True or False depending on whether the match was succesful.

import deps

deps.rule("*.o", "{p}.c", deps.c_compiler)
deps.rule("hello" + deps.executable_ext, "hello.o", deps.c_linker)

deps.build("hello" + deps.executable_ext)
